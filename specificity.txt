CSS Specificity Answers

1. Paragraph A matches only the element selector p.

It receives:
- color: gray;
- font-size: 1rem;
- font-family: Georgia, serif;
- line-height: 1.4;
- padding: 4px 0;

All these properties come from p.


2. Paragraph B matches p and .info.

From p, it keeps:
- line-height: 1.4;

From .info, it receives:
- color: steelblue;
- font-size: 1.25rem;
- background-color: #e8f4fd;
- padding: 8px 14px;
- border-left: 5px solid steelblue;
- font-family: Arial, sans-serif;

.info wins over p for the repeated properties because a class selector is stronger than an element selector.
class selector >>> element selector


3. Paragraph C matches p and #warning.

From p, it keeps:
- line-height: 1.4;

From #warning, it receives:
- color: darkred;
- font-size: 0.8rem;
- font-style: italic;
- text-transform: uppercase;
- letter-spacing: 0.15em;
- border: 2px dashed darkred;
- padding: 10px;
- background-color: #fff0f0;
- font-family: 'Courier New', monospace;

#warning wins over p because an id selector is stronger than an element selector.
id selector >>> element selector


4. Paragraph D matches all four selectors:
- p
- .info
- #warning
- #warning.info

For color, the browser uses:
color: darkorange;

This comes from #warning.info because it has the highest specificity: id + class.

For font-family, the browser uses:
font-family: 'Courier New', monospace;

This comes from #warning. #warning.info does not define font-family, so the browser keeps the value from #warning.


5. External CSS and internal CSS:

An external CSS rule is written in a separate CSS file, such as style.css.
An internal CSS rule is written inside the HTML file, in a style tag in the head.
If both external and internal CSS target the same element and property with the same specificity, the rule that appears later wins.
For example, if external CSS makes h1 brown, but internal CSS later makes h1 darkred, the browser uses darkred.
